# TOSRC 语义路由调度引擎

## 项目定位

TOSRC（Tenant-Oriented Open Semantic Routing & Control）是**100%自主可控、零外部依赖**的语义驱动路由调度引擎，采用「通用底座独立封装 + 单/多租户分项目隔离」的架构设计，聚焦本地语义识别、策略化路由、插件化场景适配核心能力，完美适配离线、内网及涉密环境。

> 重要提示：本项目**不是大模型应用**。所有语义识别能力均基于本地规则引擎 + 轻量离线模型实现，无任何联网大模型依赖，完全自主可控。

## 整体架构

| 组件 | 定位 | 适用场景 |
|------|------|----------|
| **TOSRC-Core** | 通用核心底座 | 封装所有无租户差异的通用逻辑，作为独立 Python 包供其他项目依赖 |
| **TOSRC-Single** | 单租户离线版本 | 适配楼栋机房、学校、企业内网等**离线/私有化场景**，基于 SQLite 本地存储，无网络依赖，轻量化部署 |

## 核心能力

### 1. 语义识别（NLU）
- 本地轻量规则引擎实现意图、实体解析，支持离线运行
- 情绪分析：正负向识别、强度判断、反讽识别、细分情绪类型
- 语义分类：TF-IDF + 轻量机器学习实现语义分类
- 无任何联网大模型依赖，完全自主可控

### 2. 策略路由
- 规则化、可视化配置路由策略
- 基于语义解析结果精准分发请求到插件、设备、工单等目标
- 全链路请求追溯、统计、审计

### 3. 插件化适配
- 标准化插件接口，支持本地插件加载、卸载、版本管理
- 通用插件模板，快速开发适配不同行业场景（租房、校园、园区、政务等）
- 零代码扩展新行业，仅需配置规则包即可上线

### 4. 事件总线
- 通用事件分发、订阅机制
- 支持设备事件、业务事件的联动处理
- 可扩展实现各类自动化业务流程

## 快速开始

### 单租户离线部署（TOSRC-Single）

适配内网/机房/学校等离线场景，解压即跑，无需网络：

```bash
cd TOSRC-Single
pip install -r requirements.txt
python main.py
```

访问地址：http://localhost:8080/admin/

### 核心底座开发（TOSRC-Core）

```bash
cd TOSRC-Core
pip install -e .
```

打包发布：

```bash
python -m build
twine upload dist/*
```

## 架构优势

1. **完全自主可控**：无任何第三方大模型依赖，所有能力本地实现，满足等保/合规要求
2. **场景适配精准**：单租户离线轻量化，完美匹配各类敏感场景需求
3. **维护效率极高**：通用底座独立迭代，单租户仅开发适配层，避免重复编码
4. **扩展性极强**：插件化架构，按需加载本地插件，灵活适配各类行业场景
5. **部署简单便捷**：单租户离线解压即跑，运维成本极低

## 版本说明

- **TOSRC-Core**：采用语义化版本（如 1.0.0），核心接口变更升级主版本
- **TOSRC-Single**：版本与 Core 对齐（如 1.0.0-single），仅迭代离线适配层

## 贡献指南

本项目为 monorepo 结构，请按功能提交到对应目录：

1. 通用功能请提交到 `TOSRC-Core/`
2. 单租户场景适配请提交到 `TOSRC-Single/`
3. 所有提交需符合代码规范，通过单元测试
