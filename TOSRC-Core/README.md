# TOSRC-Core 语义路由调度引擎核心底座
## 项目概述
TOSRC-Core是TOSRC架构的通用核心层，封装所有无租户差异的通用逻辑，作为独立Python包供单/多租户项目依赖引入，不包含任何租户相关、网络相关的差异化代码。
## 核心能力
### 1. 语义识别（NLU）
- 意图识别：通用规则+本地轻量模型实现意图解析
- 实体提取：通用词典+正则配置实现各类实体提取
- 语义分类：TF-IDF+轻量机器学习实现语义分类
- 情绪分析：正负向识别、强度判断、反讽识别
### 2. 路由调度
- 策略引擎：规则化、可视化配置路由策略
- 请求分发：将请求精准分发到插件、设备、工单等
- 链路管理：全链路请求追溯、统计
### 3. 插件管理
- 插件加载/卸载/版本管理
- 标准化插件接口，兼容本地/云插件
- 通用插件模板，快速开发行业插件
### 4. 事件总线
- 通用事件分发、订阅机制
- 通用事件模型（设备事件、业务事件）
## 目录结构
```plain text
src/
├── semantic/          # 语义识别核心（NLU）
│   ├── intent/        # 意图识别
│   ├── entity/        # 实体解析
│   └── classify/      # 语义分类引擎
├── router/            # 路由调度核心
│   ├── strategy/      # 路由策略引擎
│   ├── dispatcher/    # 请求分发器
│   └── link/          # 路由链路管理
├── plugin/            # 插件管理核心
│   ├── manager/       # 插件加载、卸载、升级逻辑
│   ├── interface/     # 插件标准化接口
│   └── common/        # 通用插件模板
├── event/             # 事件总线核心
│   ├── bus/           # 事件分发、订阅机制
│   └── model/         # 通用事件模型
└── common/            # 通用工具层
    ├── utils/         # 通用工具函数
    ├── config/        # 通用配置模板
    └── exception/     # 通用异常定义
```
## 安装使用
### 1. 本地开发安装
```bash
pip install -e .
```
### 2. 发布到私有PyPI
```bash
python setup.py sdist bdist_wheel
twine upload dist/* --repository-url https://your-private-pypi.com
```
### 3. 项目中引入
在requirements.txt中添加：
```
TOSRC-core>=1.0.0
```
## 版本管理
- 采用语义化版本号：主版本号.次版本号.补丁版本号
- 核心接口变更升级主版本
- 功能新增升级次版本
- Bug修复升级补丁版本
## 注意事项
- TOSRC-Core仅包含通用逻辑，不涉及任何租户、网络相关代码
- 所有差异化功能通过租户适配层实现
- 迭代时保持接口向后兼容，避免影响租户项目稳定性
